================================================================================
  Claude DevTools 困ったときは
  Troubleshooting Guide
================================================================================

このガイドでは、よくあるトラブルと、その解決方法を
イラストのようにわかりやすく説明します。

【このガイドの使い方】

  1. 症状※から該当する項目を探す
  2. 【原因】を読んで、何が問題かを理解する
  3. 【解決方法】の手順を順番に試す
  4. それでも解決しない場合は、システム管理者に相談

  ※症状：起きている問題のこと

【トラブル1】start.batをダブルクリックしても何も起きない
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  - start.batをダブルクリックしても画面が出ない
  - 一瞬黒い画面が出てすぐ消える

【原因】
  - ファイルの場所が違う
  - PowerShell※が正しくインストールされていない
  - セキュリティソフト※がブロックしている

  ※PowerShell：Windowsでコマンドを実行するためのツール
  ※セキュリティソフト：ウイルス対策ソフトなど

【解決方法】

  ▶ 手順1: ファイルの場所を確認
    1. エクスプローラー※を開く
    2. アドレスバー※に「D:\Claude-EdgeChromeDevTools」と入力
    3. start.batが見えることを確認

    ※エクスプローラー：ファイルやフォルダを見るアプリ
    ※アドレスバー：上部の場所を表示する部分

  ▶ 手順2: 右クリックで実行を試す
    1. start.batを右クリック
    2. 「管理者として実行」を選択
    3. 「ユーザーアカウント制御※」が出たら「はい」

    ※ユーザーアカウント制御：Windowsの安全確認画面

  ▶ 手順3: セキュリティソフトを確認
    → システム管理者に「start.batがブロックされていないか確認」を依頼

【トラブル2】「接続テスト失敗」と表示される
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  メニューで[3]または[4]を選ぶと、以下が表示される：
  ❌ Edge DevTools 接続テスト失敗

【原因】
  - ブラウザが既に起動している
  - 他のプログラムがポート※を使っている
  - ファイアウォール※がブロックしている

  ※ポート：ネットワーク通信の出入り口（番号で管理）
  ※ファイアウォール：不正なアクセスを防ぐ壁

【解決方法】

  ▶ 手順1: すべてのブラウザを閉じる
    1. タスクバー※のEdgeまたはChromeアイコンを右クリック
    2. 「ウィンドウを閉じる」または「すべてのウィンドウを閉じる」を選択

    ※タスクバー：画面の一番下にあるバー

    💡 完全に閉じるコツ：
    - タスクマネージャー※（Ctrl + Shift + Esc）を開く
    - 「プロセス※」タブで「Microsoft Edge」や「Google Chrome」を探す
    - すべて選択して「タスクの終了」

    ※タスクマネージャー：実行中のプログラムを管理するツール
    ※プロセス：実行中のプログラム

  ▶ 手順2: もう一度テストを実行
    1. start.batを起動
    2. メニューで[3]（Edge）または[4]（Chrome）を選択
    3. 成功メッセージが出ればOK

  ▶ 手順3: それでもダメな場合
    → パソコンを再起動※してから、もう一度試す

    ※再起動：パソコンを一度終了して、もう一度起動すること

【トラブル3】「Permission denied」や「接続が拒否されました」
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  以下のようなエラーメッセージが表示される：
  - Permission denied (publickey)
  - Connection refused
  - ssh: connect to host windows-dev port 22: Connection refused

【原因】
  - SSH接続※の設定が完了していない
  - Linuxサーバーが起動していない
  - ネットワーク※に問題がある

  ※SSH接続：離れたコンピュータに安全につなぐ方法
  ※ネットワーク：コンピュータ同士をつなぐ仕組み

【解決方法】

  ⚠️ この問題は、自分だけでは解決できません

  システム管理者に以下を伝えてください：

  「SSH接続でエラーが出ます。以下を確認してください：
   - Linuxサーバー（windows-dev）が起動しているか
   - SSH鍵※の設定が正しいか
   - ネットワークに問題がないか」

  ※SSH鍵：パスワードより安全な認証方法

【トラブル4】ブラウザが起動しない
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  - メニューで[1]または[2]を選んでも、ブラウザが開かない
  - 「ブラウザが見つかりません」とエラーが出る

【原因】
  - EdgeまたはChromeがインストールされていない
  - インストール場所が標準と違う
  - ブラウザが破損※している

  ※破損：壊れている、正常に動かない状態

【解決方法】

  ▶ 手順1: ブラウザがインストールされているか確認
    1. スタートメニューを開く
    2. 「Microsoft Edge」または「Google Chrome」を検索
    3. 見つかればインストールされている

  ▶ 手順2: 手動で起動してみる
    1. スタートメニューから「Microsoft Edge」を起動
    2. 正常に起動すればOK
    3. 起動しない場合は、ブラウザの再インストール※が必要

    ※再インストール：一度削除してから、もう一度インストールすること

  ▶ 手順3: システム管理者に相談
    「ブラウザの場所が標準と違う可能性があります。
     設定ファイル（config.json）の確認をお願いします」

【トラブル5】画面が真っ黒で何も表示されない
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  - ターミナル※画面が真っ黒
  - カーソル※（点滅する線）だけが見える
  - 何も入力できない

  ※ターミナル：文字で操作する黒い画面
  ※カーソル：文字を入力する位置を示す点滅する線

【原因】
  - 処理中で待機している
  - プログラムがフリーズ※している
  - 画面が正しく表示されていない

  ※フリーズ：固まって動かなくなること

【解決方法】

  ▶ 手順1: まず待ってみる
    1. 2〜3分そのまま待つ
    2. 何か変化があるか確認

  ▶ 手順2: Enterキーを押してみる
    1. キーボードの「Enter」キーを1回押す
    2. 何か表示されるか確認

  ▶ 手順3: 強制的に中断する
    1. キーボードの「Ctrl + C」を押す
       （「Ctrl」キーを押しながら「C」キーを押す）
    2. プログラムが終了するか確認

  ▶ 手順4: ウィンドウを閉じて再起動
    1. 右上の「×」ボタンでウィンドウを閉じる
    2. start.batを再度起動
    3. 同じ操作を試す

【トラブル6】動作が遅い・重い
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  - 起動に時間がかかる
  - Claude Codeの反応が遅い
  - パソコン全体が重く感じる

【原因】
  - 他のアプリケーションがメモリ※を使っている
  - 複数のClaude Codeセッションを開いている
  - パソコンのスペック※が不足している

  ※メモリ：コンピュータの作業机のようなもの
  ※スペック：性能、能力

【解決方法】

  ▶ 手順1: 他のアプリを閉じる
    1. タスクバーを確認
    2. 使っていないアプリケーションを閉じる
    3. 特にブラウザのタブ※を減らす

    ※タブ：ブラウザで複数のページを開く機能

  ▶ 手順2: メモリ使用量を確認
    1. タスクマネージャー（Ctrl + Shift + Esc）を開く
    2. 「パフォーマンス」タブを選択
    3. 「メモリ」の使用率を確認
    4. 90%以上なら、アプリを閉じてメモリを解放※

    ※解放：使っていたメモリを空ける

  ▶ 手順3: パソコンを再起動
    1. すべての作業を保存
    2. 「スタート」→「電源」→「再起動」
    3. 再起動後、Claude DevToolsのみを起動

【トラブル7】プロジェクトが表示されない
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  プロジェクト選択画面で、何も表示されない
  または
  「Zドライブにプロジェクトが見つかりません」と表示される

【原因】
  - Zドライブ※がマッピング※されていない
  - プロジェクトフォルダがZドライブにない
  - 設定ファイル（config.json）の設定が違う

  ※Zドライブ：ネットワーク上のフォルダをドライブとして見せる機能
  ※マッピング：ネットワーク上のフォルダをドライブ文字（Z:など）で使えるようにすること

【解決方法】

  ▶ 手順1: Zドライブがあるか確認
    1. エクスプローラーを開く
    2. 左側の「PC」をクリック
    3. 「Z:」ドライブが表示されているか確認

  ▶ 手順2: Zドライブを開いてみる
    1. 「Z:」をダブルクリック
    2. フォルダが表示されるか確認
    3. エラーが出る場合は、ネットワーク接続に問題あり

  ▶ 手順3: システム管理者に相談
    「Zドライブが見えません」または
    「Zドライブは見えますが、プロジェクトがありません」
    と伝える

【トラブル8】Claude Codeがすぐに終了してしまう
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  - Claude Codeが起動してもすぐに終了する
  - 「🔄 Claude 再起動 (3秒後)...」と繰り返し表示される

【原因】
  - Claude CLIツール※がLinux側にインストールされていない
  - 環境変数※が正しく設定されていない
  - Linuxサーバーの問題

  ※CLIツール：コマンドラインインターフェースツール（文字で操作するツール）
  ※環境変数：プログラムが動くための設定情報

【解決方法】

  ⚠️ この問題は、Linuxサーバー側の設定が必要です

  システム管理者に以下を伝えてください：

  「Claude Codeがすぐに終了します。
   - Claude CLIがインストールされているか確認をお願いします
   - コマンド: which claude
   - バージョン確認: claude --version」

【トラブル9】エラーメッセージの意味がわからない
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【症状】
  英語や専門用語のエラーメッセージが表示される

【解決方法】

  ▶ 手順1: スクリーンショット※を撮る
    1. キーボードの「Windowsキー + Shift + S」を押す
    2. マウスで範囲を選択
    3. クリップボード※に保存される

    ※スクリーンショット：画面をそのまま画像として保存すること
    ※クリップボード：コピーしたものを一時的に保存する場所

  ▶ 手順2: エラーメッセージをメモ
    マウスでエラーメッセージを選択して、
    右クリック→「コピー」でテキスト※として保存

    ※テキスト：文字情報

  ▶ 手順3: システム管理者に相談
    撮影したスクリーンショットまたはコピーしたテキストを見せて相談
