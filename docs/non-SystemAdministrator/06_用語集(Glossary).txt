================================================================================
  Claude DevTools 用語集
  Glossary
================================================================================

このドキュメントでは、Claude DevToolsを使う際に出てくる
専門用語を、わかりやすく説明します。

【用語の見方】

  用語名（読み方）
    説明文
    （例：具体的な使用例）

■AI（エーアイ）

  人工知能のこと。コンピュータが人間のように考えて、
  質問に答えたり、作業を手伝ったりする技術。

  （例：Claude CodeはAIを使ったプログラミング支援ツール）

■API（エーピーアイ）

  Application Programming Interfaceの略。
  プログラム同士が情報をやりとりする仕組み。

  （例：Claude CodeはAPIを通じてブラウザと通信する）

■Bash（バッシュ）

  Linux/Unixで使われるコマンド入力ツール（シェル）の一種。
  文字で指示を出すためのプログラム。

  （例：run-claude.shはBashスクリプト）

■Chrome（クローム）

  Googleが提供している無料のWebブラウザ。
  正式名称はGoogle Chrome。

  （例：Claude DevToolsはChromeに対応している）

■CLI（シーエルアイ）

  Command Line Interfaceの略。
  文字（コマンド）で操作するインターフェース。

  （例：ClaudeコマンドはCLIツール）

■CPU（シーピーユー）

  Central Processing Unitの略。
  コンピュータの頭脳に当たる部品。計算や処理を行う。

  （例：CPUの性能が高いほど、処理が速い）

■DevTools（デブツールズ）

  Developer Toolsの略。開発者ツール。
  ブラウザに内蔵されている、プログラム開発用の機能。

  （例：F12キーを押すとDevToolsが開く）

■Edge（エッジ）

  Microsoftが提供している無料のWebブラウザ。
  正式名称はMicrosoft Edge。Windows 10/11に最初から入っている。

  （例：Claude DevToolsはEdgeに対応している）

■Agent Teams（エージェントチームズ）

  Claude Codeの実験的な機能。
  複数のAIアシスタントがチームとして一緒に作業できる仕組み。

  （例：Agent Teamsを使って複数のタスクを同時に進める）

■base64（ベースろくよん）

  データを安全に転送するための文字変換方式。
  特殊な文字を含むデータも問題なく送信できる。

  （例：スクリプトをbase64でエンコードしてSSH転送する）

■Git（ギット）

  プログラムの変更履歴を記録・管理するツール。
  バージョン管理システム。

  （例：Gitブランチは作業の枝分かれを表す）

■heredoc（ヒアドキュメント）

  プログラミングで長い文章を安全に書くための方法。
  特殊な文字も問題なく含められる。

  （例：初期プロンプトはheredoc方式で定義されている）

■GitHub（ギットハブ）

  Gitを使ったプログラムの共有サービス。
  世界中の開発者が使っている。

  （例：GitHubでコードを公開・共有できる）

■IP（アイピー）

  Internet Protocolの略。
  ネットワーク上の住所のようなもの。

  （例：192.168.1.100はIPアドレス）

■Linux（リナックス）

  Windowsとは別の種類のOS（オペレーティングシステム）。
  サーバーでよく使われる。

  （例：このツールではLinuxサーバーでClaude Codeを実行する）

■OS（オーエス）

  Operating Systemの略。基本ソフトウェア。
  コンピュータ全体を管理するプログラム。

  （例：WindowsやLinuxはOSの一種）

■PowerShell（パワーシェル）

  Windowsで使われる高機能なコマンド入力ツール。
  複雑な作業を自動化できる。

  （例：このツールのメインスクリプトはPowerShellで書かれている）

■RAM（ラム）

  Random Access Memoryの略。メモリのこと。
  コンピュータの作業机のようなもの。

  （例：RAMが多いほど、同時にたくさんの作業ができる）

■SSH（エスエスエイチ）

  Secure Shellの略。
  離れたコンピュータに安全につなぐための仕組み。

  （例：SSHでLinuxサーバーに接続する）

■Statusline（ステータスライン）

  画面の下部に表示される状態表示バー。
  現在のフォルダ、Gitブランチ、AIモデル名などを表示する。

  （例：StatuslineでAIのメモリ使用率を確認する）

■URL（ユーアールエル）

  Uniform Resource Locatorの略。
  インターネット上の住所。

  （例：https://www.google.com はURLの一例）

■USB（ユーエスビー）

  Universal Serial Busの略。
  パソコンと周辺機器をつなぐ規格。

  （例：USBメモリはUSB接続の記憶装置）

■アカウント

  サービスやシステムを使うための登録情報。
  ユーザー名とパスワードで本人確認する。

  （例：Googleアカウント、Microsoftアカウント）

■アップデート

  ソフトウェアを最新版に更新すること。
  新機能の追加やバグ修正が含まれる。

  （例：月に1回、アップデートを実行する）

■アドレスバー

  ブラウザの上部にある、URLを表示・入力する部分。

  （例：アドレスバーに「google.com」と入力してEnter）

■暗号化（あんごうか）

  情報を他人が読めないように変換すること。
  セキュリティ対策の一つ。

  （例：パスワードは暗号化して保存される）

■インストール

  ソフトウェアをパソコンに入れること。
  セットアップともいう。

  （例：Windows Terminalをインストールする）

■インターフェース

  人とコンピュータの接点。操作画面や方法。

  （例：グラフィカルユーザーインターフェース（GUI）は
        マウスで操作できる画面）

■ウィンドウ

  画面に表示される四角い枠。
  アプリケーションの表示領域。

  （例：ブラウザのウィンドウを閉じる）

■エクスプローラー

  Windowsでファイルやフォルダを見るためのアプリ。
  ファイルマネージャー。

  （例：エクスプローラーでDドライブを開く）

■エラー

  間違い、不具合、失敗。
  プログラムが正しく動かない状態。

  （例：エラーメッセージが表示される）

■エンコーディング

  文字をコンピュータで扱うための変換方式。

  （例：UTF-8は日本語を含む多言語に対応したエンコーディング）

■オープンソース

  プログラムの中身（ソースコード）が公開されていて、
  誰でも自由に使える・改造できるソフトウェア。

  （例：このツールはオープンソースのMITライセンス）

■解像度（かいぞうど）

  画面の細かさ。ピクセル数で表す。

  （例：1920×1080は横1920、縦1080ピクセルの解像度）

■拡張子（かくちょうし）

  ファイル名の最後に付く「.txt」「.exe」などの部分。
  ファイルの種類を表す。

  （例：start.batの「.bat」は拡張子）

■カーソル

  文字を入力する位置を示す点滅する線。

  （例：カーソルを移動して文字を挿入する）

■管理者（かんりしゃ）

  システムを管理・運用する人。
  特別な権限を持つユーザー。

  （例：管理者権限で実行する）

■起動（きどう）

  プログラムやコンピュータを開始すること。

  （例：start.batを起動する）

■キャッシュ

  よく使うデータを一時的に保存して、
  次回アクセスを速くする仕組み。

  （例：ブラウザのキャッシュをクリアする）

■クラウド

  インターネット上で提供されるサービス。
  自分のパソコンではなく、インターネット上のサーバーを使う。

  （例：クラウドストレージにファイルを保存）

■クリップボード

  コピーしたものを一時的に保存する場所。

  （例：クリップボードにコピーして貼り付ける）

■クローン

  複製、コピー。
  Gitでは、リポジトリ全体をコピーすること。

  （例：GitHubからリポジトリをクローンする）

■コマンド

  コンピュータに指示を出すための文字列。
  命令文。

  （例：「exit」はプログラムを終了するコマンド）

■コマンドプロンプト

  Windowsで文字入力により操作する画面。
  黒い背景に白い文字。

  （例：コマンドプロンプトでコマンドを実行する）

■コピー＆ペースト

  文字や画像をコピーして、別の場所に貼り付けること。
  Ctrl+C（コピー）とCtrl+V（ペースト）。

  （例：エラーメッセージをコピー＆ペースト）

■再起動（さいきどう）

  パソコンやプログラムを一度終了して、
  もう一度起動すること。

  （例：問題が起きたらパソコンを再起動する）

■サーバー

  他のコンピュータにサービスを提供する
  専用のコンピュータ。

  （例：Linuxサーバーでプログラムを実行）

■サブフォルダ

  フォルダの中にあるフォルダ。
  階層構造を作る。

  （例：Dドライブ → Claude-EdgeChromeDevTools → docs）

■シェル

  コマンドを入力してコンピュータを操作するツール。

  （例：BashやPowerShellはシェルの一種）

■システム

  複数の要素が組み合わさって動く仕組み。

  （例：オペレーティングシステム、ファイルシステム）

■ショートカット

  ファイルやフォルダに素早くアクセスするための目印。
  実体ではなく、参照（リンク）。

  （例：デスクトップにショートカットを作る）

■ショートカットキー

  キーボードの組み合わせで素早く操作する方法。

  （例：Ctrl+C はコピーのショートカットキー）

■スクリーンショット

  画面をそのまま画像として保存すること。
  画面キャプチャ。

  （例：Windowsキー+Shift+S でスクリーンショット）

■スクリプト

  自動実行できるプログラム。
  複数のコマンドをまとめたもの。

  （例：start.batはバッチスクリプト）

■ステータスライン

  状態を表示するバー。
  現在の情報を表示する行。

  （例：Claude Codeのステータスラインにプロジェクト名が表示）

■セキュリティ

  情報を守るための安全対策。
  不正アクセスやウイルスから保護する。

  （例：セキュリティソフトを最新に保つ）

■セッション

  作業の期間。
  接続している状態。

  （例：Claude Codeのセッションを終了する）

■ソフトウェア

  プログラムのこと。
  ハードウェア（物理的な機器）の対義語。

  （例：Windows TerminalはソフトウェアのWindowsはソフトウェアの一種）

■ターミナル

  文字入力で操作する画面。
  コマンドライン環境。

  （例：Windows Terminalは高機能なターミナル）

■ダウンロード

  インターネットからファイルを自分のパソコンに
  取り込むこと。

  （例：ブラウザからファイルをダウンロード）

■タスクバー

  Windowsの画面の一番下（または上・横）にあるバー。
  起動中のアプリや時刻が表示される。

  （例：タスクバーからアプリを起動）

■タスクマネージャー

  実行中のプログラムを管理するツール。
  Ctrl+Shift+Escで起動。

  （例：タスクマネージャーでメモリ使用率を確認）

■ダブルクリック

  マウスの左ボタンを素早く2回押すこと。
  ファイルやフォルダを開く操作。

  （例：start.batをダブルクリックして起動）

■チェックボックス

  選択肢を複数選べる四角いボックス。
  チェックマーク（✓）を入れる。

  （例：チェックボックスにチェックを入れる）

■ディレクトリ

  フォルダのこと。
  ファイルを整理するための入れ物。

  （例：プロジェクトディレクトリを開く）

■デフォルト

  初期設定。
  何も選ばなかった時に自動で選ばれる選択肢。

  （例：デフォルトのブラウザはEdge）

■ドライブ

  パソコンの中のデータ保存場所。
  C:、D:、Z:などの文字で識別。

  （例：Dドライブにプロジェクトがある）

■ドラッグ＆ドロップ

  マウスでアイテムをつかんで移動する操作。

  （例：ファイルをドラッグ＆ドロップで移動）

  情報をやりとりする仕組み。

  （例：社内ネットワークに接続）

■パーミッション

  権限、許可。
  ファイルやフォルダへのアクセス権。

  （例：読み取り権限、書き込み権限）

■バージョン

  ソフトウェアの版、世代。
  更新されるたびに番号が上がる。

  （例：バージョン1.0.0から1.1.0へアップデート）

■バックアップ

  データのコピーを作って保管すること。
  万が一の際の保険。

  （例：重要なファイルはバックアップを取る）

■パス

  ファイルやフォルダの場所を示す文字列。
  道筋、経路。

  （例：D:\Claude-EdgeChromeDevTools\start.bat）

■パスワード

  本人確認のための秘密の文字列。

  （例：パスワードは他人に教えてはいけない）

■ハードウェア

  物理的な機器、装置。
  パソコン本体やキーボード、マウスなど。

  （例：CPUやメモリはハードウェアの一部）

■ファイアウォール

  不正なアクセスを防ぐ壁。
  セキュリティ機能の一つ。

  （例：ファイアウォールでポートを許可する）

■ファイル

  データをまとめたもの。
  文書、画像、プログラムなど。

  （例：start.batはバッチファイル）

■フォルダ

  ファイルを整理するための入れ物。
  ディレクトリともいう。

  （例：docsフォルダにドキュメントを保存）

■フォント

  文字の形、書体。

  （例：Cascadia Codeは等幅フォント）

■ブラウザ

  インターネットを見るためのソフト。
  Webブラウザ。

  （例：EdgeやChromeはブラウザ）

■ブランチ

  Gitでの作業の枝分かれ。
  バージョンの派生。

  （例：mainブランチは本流）

■プログラム

  コンピュータに指示を出すための命令文の集まり。
  ソフトウェア。

  （例：このツールはPowerShellプログラム）

■プロセス

  実行中のプログラム。

  （例：タスクマネージャーでプロセスを確認）

■プロジェクト

  作業を行う場所、単位。
  ここではフォルダのこと。

  （例：my-projectフォルダで作業する）

■プロトコル

  通信の規則、手順。

  （例：HTTPはWebページを見るためのプロトコル）

■ポート

  ネットワーク通信の出入り口を識別する番号。

  （例：ポート9222でDevToolsに接続）

  RAM。作業机のようなもの。

  （例：メモリが不足すると動作が遅くなる）

  （例：このツールはMITライセンス）

■リソース

  資源。
  コンピュータの処理能力やメモリなど。

  （例：リソースを効率的に使う）

■リモート

  遠隔、離れた場所。

  （例：リモートのLinuxサーバーに接続）

■ログ

  記録、履歴。
  何が起きたかの記録。

  （例：エラーログを確認する）

■ローカル

  自分のパソコン。
  リモートの対義語。

  （例：ローカルのファイルを編集）
