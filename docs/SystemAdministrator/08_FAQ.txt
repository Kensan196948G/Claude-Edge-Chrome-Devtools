================================================================================
  Claude-EdgeChromeDevTools FAQ（よくある質問）
  Frequently Asked Questions
================================================================================

【一般的な質問】

Q1: このプロジェクトの目的は何ですか？
A1: WindowsマシンとリモートLinuxホスト間でClaude Codeとブラウザ開発者ツール
    (DevTools)を統合し、リモート開発環境を迅速にセットアップすることです。

Q2: どのブラウザがサポートされていますか？
A2: Microsoft EdgeとGoogle Chromeがサポートされています。Chromium系ブラウザ
    （Brave、Operaなど）も動作する可能性がありますが、公式にはサポート外です。

Q3: Linuxホストは必須ですか？
A3: はい、このプロジェクトはリモートLinuxホスト上でClaude Codeを実行することを
    前提としています。ローカルWindowsのみでの実行は現在サポートしていません。

Q4: 複数のプロジェクトを同時に実行できますか？
A4: はい、config.jsonのports配列に十分なポート数を設定すれば、複数プロジェクトを
    同時に実行できます。各プロジェクトは異なるポートを使用します。

Q5: 商用プロジェクトで使用できますか？
A5: はい、MIT Licenseの下で自由に使用できます。詳細は10_ライセンス(License).txt
    を参照してください。

【セットアップに関する質問】

Q6: Zドライブが必須ですか？別のドライブは使えませんか？
A6: Zドライブは必須ではありません。config.jsonのzDrive設定を任意のパスに変更
    できます。例: "zDrive": "D:\\Projects\\"

Q7: SSHキーの設定方法を教えてください
A7: 以下の手順で設定できます：
    1. ssh-keygen -t rsa -b 4096 -C "your-email@example.com"
    2. ssh-copy-id windows-dev
    3. ssh windows-dev でパスワードなしで接続できることを確認

Q8: Windows Terminalは必須ですか？
A8: 必須ではありませんが、強く推奨します。通常のコマンドプロンプトやPowerShell
    でも動作しますが、Windows Terminalの方が表示が綺麗で使いやすいです。

Q9: jqがインストールされていません。自動インストールされますか？
A9: はい、Statusline機能が有効な場合、スクリプトがjqの自動インストールを試行
    します。失敗した場合は手動でインストールしてください：
    sudo apt-get install jq  # Ubuntu/Debian
    sudo yum install jq      # CentOS/RHEL

【使用方法に関する質問】

Q10: 初期プロンプトをカスタマイズできますか？
A10: はい、scripts/main/Claude-EdgeDevTools.ps1（または Claude-ChromeDevTools-Final.ps1）
     の296行目付近にあるINIT_PROMPT変数を編集してください。

Q11: Statuslineの表示をカスタマイズできますか？
A11: はい、scripts/statusline.sh を編集することで独自の表示を追加できます。
     Bashスクリプトの知識が必要です。

Q12: 特定のポートを強制的に使用するにはどうすればよいですか？
A12: config.jsonのports配列で、希望するポートを最初に配置してください。
     例: "ports": [9225, 9222, 9223, 9224]  # 9225が優先

Q13: プロジェクトの切り替えは簡単ですか？
A13: はい、start.batを実行するたびにプロジェクト選択画面が表示されるので、
     簡単に切り替えられます。

Q14: Claude Codeセッション中にStatuslineを更新するには？
A14: Claude Code内で `/statusline` コマンドを実行してください。

【トラブルシューティングに関する質問】

Q15: DevTools接続が失敗します
A15: 以下を確認してください：
     1. すべてのブラウザウィンドウを閉じる
     2. http://localhost:9222/json/version にアクセスできるか確認
     3. ファイアウォールでポート9222-9225が許可されているか確認
     詳細は05_トラブルシューティング(Troubleshooting).txt を参照

Q16: SSH接続でPermission deniedエラーが出ます
A16: SSHキーが正しく設定されているか確認してください：
     1. ~/.ssh/config に windows-dev が定義されているか
     2. ssh windows-dev でパスワードなしで接続できるか
     3. ssh -vvv windows-dev で詳細ログを確認

Q17: Statuslineが表示されません
A17: 以下を試してください：
     1. Claude Code内で `/statusline` コマンド実行
     2. Claude Codeを終了して再起動
     3. jqがインストールされているか確認: which jq
     4. statusline.shに実行権限があるか確認: ls -l ~/.claude/statusline.sh

Q18: 「利用可能なポートがありません」エラーが出ます
A18: config.jsonのports配列にポートを追加してください：
     "ports": [9222, 9223, 9224, 9225, 9226, 9227, 9228, 9229, 9230]
     また、不要なブラウザプロセスを終了してください。

Q19: ブラウザが起動しません
A19: config.jsonのedgeExe/chromeExeパスが正しいか確認してください。
     標準インストール先と異なる場合は、実際のパスに変更する必要があります。

【パフォーマンスに関する質問】

Q20: 起動が遅いです。高速化できますか？
A20: 以下を試してください：
     - SSDを使用する
     - ブラウザプロファイルをRAMディスクに配置する
     - Statusline機能を無効化する（わずかな効果）
     - 不要なブラウザ拡張機能を無効化する

Q21: メモリ使用量を削減できますか？
A21: 以下が効果的です：
     - 不要なタブを閉じる
     - ブラウザ拡張機能を最小限にする
     - 複数プロジェクトの同時実行を避ける

【セキュリティに関する質問】

Q22: --dangerously-skip-permissions フラグは安全ですか？
A22: 開発環境でのみ使用してください。本番環境や信頼できないコードでは使用
     しないでください。このフラグはClaude Codeが権限確認なしでファイル操作を
     行えるようにします。

Q23: SSH接続は暗号化されていますか？
A23: はい、SSH接続は暗号化されています。SSHトンネルを通じてDevToolsプロトコルも
     安全に転送されます。

Q24: ブラウザプロファイルに機密情報は保存されますか？
A24: 専用プロファイル（C:\DevTools-{browser}-{port}）を使用しているため、通常の
     ブラウジングデータとは分離されています。ただし、開発中のプロジェクトで
     扱う情報には注意してください。

【高度な使用方法に関する質問】

Q25: カスタムブラウザ（例：Firefox）を追加できますか？
A25: はい、以下の手順で追加できます：
     1. config.jsonに設定追加
     2. 新しいメインスクリプト作成（Claude-FirefoxDevTools.ps1）
     3. start.batにメニュー追加
     詳細は06_開発ガイド(Development-Guide).txt を参照

Q26: 環境変数をカスタマイズできますか？
A26: はい、run-claude.sh生成部分を編集することで、独自の環境変数を追加できます。
     scripts/main/Claude-EdgeDevTools.ps1 の320行目付近を参照してください。

Q28: Agent Teams機能とは何ですか？
A28: Claude Code の実験的機能で、複数のClaude Codeインスタンスが
     チームとして協調して作業できる機能です。Team Lead、Teammates、
     Task List、Mailboxの4つのコンポーネントで構成されます。
     環境変数 CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 で有効化され、
     このプロジェクトのスクリプトでは自動的に設定されます。

Q29: config.jsonのclaudeCodeセクションの役割は？
A29: Claude Codeの環境変数とUI設定を中央管理するためのセクションです。
     スクリプト実行時に、ここで定義された設定がLinux側の
     ~/.claude/settings.json に自動的にマージされます。
     既存のpermissions、plugins、hooksは保持されます。

Q30: CI/CD環境で使用できますか？
A30: 対話型UIを使用しているため、そのままではCI/CDに適していません。
     非対話型モードの実装が必要です（将来の拡張候補）。

Q31: Docker環境でも動作しますか？
A31: LinuxホストがDocker内で実行されている場合、ポートフォワーディングの
     設定を調整すれば動作する可能性がありますが、公式にはサポートしていません。

【その他の質問】

Q32: エラーログはどこに保存されますか？
A32: 現在、ログファイルへの自動保存機能はありません。Windows Terminalの
     出力を手動で保存（Ctrl+Shift+S）するか、リダイレクトを使用してください：
     start.bat > output.log 2>&1

Q33: アップデート方法は？
A33: Git経由でアップデートできます：
     git pull origin main
     設定ファイル（config.json）は変更されないので、既存の設定が保持されます。

Q34: バグを報告したいのですが
A34: プロジェクトリポジトリのIssueセクションで報告してください。以下の情報を
     含めると解決が早くなります：
     - 詳細な症状
     - 再現手順
     - 環境情報（OS、PowerShellバージョン、ブラウザバージョン）
     - エラーメッセージ

Q35: 機能リクエストはできますか？
A35: はい、GitHub Issueまたは Discussions で提案してください。
     貢献も歓迎します！

Q36: ドキュメントが古い場合はどうすればよいですか？
A36: GitHub Issueで報告するか、Pull Requestでドキュメント修正を提案してください。

Q37: このFAQに載っていない質問があります
A37: 以下のリソースを確認してください：
     - docs/フォルダ内の他のドキュメント
     - CLAUDE.md
     - GitHub Discussions
     - プロジェクトのIssue履歴
